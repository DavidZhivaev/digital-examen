# digital-examen

Система ГБОУ Школы 1580 для проведения семестровых работ с использованием цифровых технологий.
Все API-эндпоинты начинаются с `/api/`.

## Авторизация

Защищённые эндпоинты требуют заголовок:

```
Authorization: Bearer <access_token>
```

Роли (`@min_perms(N)` — доступ при `role >= N`):

| role | Название |
|------|----------|
| 1    | Учащийся |
| 2    | Учитель (классный руководитель — если прикреплён к классу как `teacher`) |
| 3    | Оператор (технический специалист) |
| 4    | Администратор |

В путях пользователей используется публичный **`person_id`** (UUID), а не числовой `id`.

## Auth — `/api/auth`

| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| `GET` | `/api/auth/health` | Публичный | Статус сервиса авторизации |
| `POST` | `/api/auth/login` | Публичный | Вход. Возвращает `access_token` и `refresh_token` |
| `POST` | `/api/auth/refresh` | Публичный | Обновление пары токенов по `refresh_token` |
| `POST` | `/api/auth/logout` | Публичный | Выход (отзыв сессии по `refresh_token`) |
| `POST` | `/api/auth/set-password` | Публичный | Установка/смена пароля по ссылке из письма |
| `GET` | `/api/auth/sessions` | `@min_perms(1)` | Список своих сессий |
| `DELETE` | `/api/auth/sessions/{session_id}` | `@min_perms(1)` | Отозвать свою сессию |
| `POST` | `/api/auth/logout-all` | `@min_perms(1)` | Отозвать все свои сессии |
| `GET` | `/api/auth/sessions/user/{user_id}` | Оператор+ | Список сессий пользователя |
| `DELETE` | `/api/auth/sessions/user/{user_id}` | Оператор+ | Отозвать все сессии пользователя |

### `POST /api/auth/login`

```json
{
  "login": "admin",
  "password": "admin12345",
  "device_name": "Chrome / Windows"
}
```

### `POST /api/auth/refresh` / `POST /api/auth/logout`

```json
{
  "refresh_token": "<refresh_token>"
}
```

## Users — `/api/users`

| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| `GET` | `/api/users/health` | Публичный | Статус сервиса пользователей |
| `GET` | `/api/users/me` | `@min_perms(1)` | Текущий пользователь |
| `PATCH` | `/api/users/me` | `@min_perms(1)` | Обновить свой профиль (без смены роли) |
| `GET` | `/api/users/` | Админ | Список пользователей с пагинацией |
| `GET` | `/api/users/{person_id}` | Админ | Пользователь по `person_id` |
| `POST` | `/api/users/` | Учитель+ | Создать пользователя (учитель — только учеников своего класса) |
| `PATCH` | `/api/users/{person_id}` | Админ | Обновить пользователя |
| `PATCH` | `/api/users/{person_id}/role` | Админ | Изменить права (роль) пользователя |
| `POST` | `/api/users/{person_id}/force-password-reset` | Оператор+ | Отправить ссылку на смену пароля (вход заблокирован) |
| `DELETE` | `/api/users/{person_id}` | Админ | Удалить пользователя |

### Пагинация — `GET /api/users/`

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `page` | `1` | Номер страницы |
| `page_size` | `20` | Размер страницы (макс. `100`) |

### `PATCH /api/users/{person_id}/role`

```json
{
  "role": 2
}
```

Админ может назначать роли только **не выше своей**, не может менять свою роль и не может управлять пользователями с равными или более высокими правами.

### Типы работ — конфиг `works/work_types.json`

Путь настраивается через `WORK_TYPES_PATH` в `.env`. Формат:

```json
{
  "math_oge": {
    "name": "ГИА-9-МАТ",
    "questions": {
      "1": "0123456789-,",
      "2": "0123456789-,"
    }
  },
  "no_test": {
    "name": "Без тестовой части",
    "questions": {}
  }
}
```

- `questions` — словарь «номер вопроса → допустимые символы ответа»
- пустой `{}` — работа без тестовой части
- предмет (`math` / `physics` / `russian`) выбирается отдельно при создании работы

### `POST /api/works/`

```json
{
  "person_ids": ["uuid-ученика-1", "uuid-ученика-2"],
  "work_type_id": "math_oge",
  "subject": "math",
  "conduct_date": "2026-06-20",
  "room_ids": [1, 2],
  "supervisor_person_ids": ["uuid-учителя-1"]
}
```

**Права при создании:**

- **Учитель** — только учащиеся одного класса
- **Оператор+** — учащиеся из нескольких классов (глобальная работа; участникам уходит email)

Если указаны `room_ids`, требуется рассадка и нужны `supervisor_person_ids`.

### Видимость работ

| Роль | Доступ |
|------|--------|
| Учащийся | Свои работы; в рассадке — только своё место (`my_seat`) |
| Классный руководитель | Работы своего класса и глобальные с его классом; полная рассадка |
| Оператор+ | Все работы |

## Безопасность

- JWT **RS256** с полями `iss`, `aud`, `iat`, `nbf`, `exp`.
- Access-токен привязан к сессии (`sid`) — при отзыве сессии токен перестаёт работать.
- Refresh-токен хранится в БД только как SHA-256 хеш.
- При повторном использовании старого refresh-токена отзываются **все** сессии пользователя.
- Не более **5** активных сессий на пользователя.
- Смена пароля или роли отзывает все сессии пользователя.
- Rate limit:
  - **10 000** запросов/мин на IP
  - **3 000** запросов/мин на авторизованного пользователя
  - **60** попыток входа/мин на IP
- `/health` и документация не лимитируются.
- Задержка ответа при неверном пароле.
- Логируются только запросы с валидным access-токеном.

Лимиты настраиваются в `.env` (`RATE_LIMIT_*`, `LOGIN_RATE_LIMIT_*`).

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env
python -m uvicorn main:app --reload
```

Первый администратор:

```bash
python -m scripts.create_admin
```

Docker:

```bash
docker compose up -d --build
docker compose exec api python -m scripts.create_admin
```

Документация Swagger (только при `DEBUG=true`): `http://localhost:8000/docs`
