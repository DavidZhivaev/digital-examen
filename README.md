# digital-examen

Система ГБОУ Школы 1580 для проведения семестровых работ с использованием цифровых технологий.
Все API-эндпоинты начинаются с `/api/`.

## Авторизация

Защищённые эндпоинты требуют заголовок:


```

Authorization: Bearer <access_token>

```

Роли:

| role | Название |
|------|----------|
| 1    | Учащийся |
| 2    | Учитель (классный руководитель — если прикреплён к классу как `teacher`) |
| 3    | Оператор (технический специалист) |
| 4    | Администратор |

## Auth — `/api/auth`

| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| `GET` | `/api/auth/health` | Публичный | Статус сервиса авторизации |
| `POST` | `/api/auth/login` | Публичный | Вход. Возвращает `access_token` и `refresh_token` |
| `POST` | `/api/auth/refresh` | Публичный | Обновление пары токенов по `refresh_token` |
| `POST` | `/api/auth/logout` | Публичный | Выход (отзыв сессии по `refresh_token`) |
| `POST` | `/api/auth/set-password` | Публичный | Установка/смена пароля по ссылке из письма |
| `GET` | `/api/auth/sessions` | `@min_perms(1)` | Список своих сессий |
| `DELETE` | `/api/auth/sessions/{session_id}` | `@min_perms(1)` | Отозвать свою сессию |
| `POST` | `/api/auth/logout-all` | `@min_perms(1)` | Отозвать все свои сессии |
| `GET` | `/api/auth/sessions/user/{user_id}` | Оператор+ | Список сессий пользователя |
| `DELETE` | `/api/auth/sessions/user/{user_id}` | Оператор+ | Отозвать все сессии пользователя |

### `POST /api/auth/login`

```json
{
  "login": "admin",
  "password": "admin12345",
  "device_name": "Chrome / Windows"
}

```

### `POST /api/auth/refresh` / `POST /api/auth/logout`

```json
{
  "refresh_token": "<refresh_token>"
}

```

## Users — `/api/users`

| Метод | Путь | Доступ | Описание |
| --- | --- | --- | --- |
| `GET` | `/api/users/health` | Публичный | Статус сервиса пользователей |
| `GET` | `/api/users/me` | `@min_perms(1)` | Текущий пользователь |
| `PATCH` | `/api/users/me` | `@min_perms(1)` | Обновить свой профиль (без смены роли) |
| `GET` | `/api/users/` | Админ | Список пользователей с пагинацией |
| `GET` | `/api/users/{user_id}` | Админ | Пользователь по ID |
| `POST` | `/api/users/` | Админ | Создать пользователя |
| `PATCH` | `/api/users/{user_id}` | Админ | Обновить пользователя |
| `PATCH` | `/api/users/{user_id}/role` | Админ | Изменить права (роль) пользователя |
| `POST` | `/api/users/{user_id}/force-password-reset` | Оператор+ | Отправить ссылку на смену пароля (вход заблокирован) |
| `DELETE` | `/api/users/{user_id}` | Админ | Удалить пользователя |

### Пагинация — `GET /api/users/`

Query-параметры:

| Параметр | По умолчанию | Описание |
| --- | --- | --- |
| `page` | `1` | Номер страницы |
| `page_size` | `20` | Размер страницы (макс. `100`) |

Ответ:

```json
{
  "items": [ ... ],
  "total": 42,
  "page": 1,
  "page_size": 20,
  "pages": 3
}

```

### `PATCH /api/users/{user_id}/role`

```json
{
  "role": 2
}

```

Админ может назначать роли только **не выше своей**, не может менять свою роль и не может управлять пользователями с равными или более высокими правами.

| `POST` | `/api/classes/{class_id}/import` | Оператор+ | Массовый импорт/создание учеников класса из Excel (`.xlsx`) |

### `POST /api/classes/`

```json
{
  "parallel": 10,
  "litera": "А",
  "corpus": 1,
  "teacher_id": 5
}

```

Класс уникален по сочетанию `parallel` + `litera` + `corpus`.
**Классный руководитель** — пользователь с ролью `2` (Учитель), назначенный полем `teacher` у класса.
Обычный учитель без прикрепленного класса не может управлять контингентом чужих классов.

## Безопасность

* JWT **RS256** с полями `iss`, `aud`, `iat`, `nbf`, `exp`.
* Access-токен привязан к сессии (`sid`) — при отзыве сессии токен мгновенно перестаёт работать.
* Refresh-токен хранится в БД только как SHA-256 хеш.
* При повторном использовании старого refresh-токена срабатывает защита от кражи токенов и отзываются **все** сессии пользователя.
* Не более **5** активных сессий на пользователя (старые автоматически удаляются при превышении лимита).
* Смена пароля или роли принудительно отзывает все сессии пользователя.
* Rate limit на все запросы (настроены под школьную нагрузку):
* **10 000** запросов/мин на IP (для корректной работы за школьным NAT).
* **3 000** запросов/мин на авторизованного пользователя.
* **60** попыток входа/мин на IP для эндпоинта логина.
* `/health` и интерактивная документация не лимитируются.


* Искусственная задержка ответа при неверном пароле для защиты от брутфорса.
* Логируются только запросы с валидным access-токеном.

Лимиты настраиваются в `.env` (`RATE_LIMIT_*`, `LOGIN_RATE_LIMIT_*`).

```bash
pip install -r requirements.txt
cp .env.example .env
python -m uvicorn main:app --reload

```

Первый администратор:

```bash
python -m scripts.create_admin

```

Docker:

```bash
docker compose up -d --build
docker compose exec api python -m scripts.create_admin

```
