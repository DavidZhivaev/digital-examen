# digital-examen

Система ГБОУ Школы 1580 для проведения семестровых работ с использованием цифровых технологий.
Все API-эндпоинты начинаются с `/api/`.

## Авторизация

Защищённые эндпоинты требуют заголовок:

```
Authorization: Bearer <access_token>
```

Роли (`@min_perms(N)` — доступ при `role >= N`):

| role | Название |
|------|----------|
| 1    | Учащийся |
| 2    | Учитель (классный руководитель — если прикреплён к классу как `teacher`) |
| 3    | Оператор (технический специалист) |
| 4    | Администратор |

В путях пользователей используется публичный **`person_id`** (UUID), а не числовой `id`.

**Права при создании:**

- **Учитель** — только учащиеся одного класса
- **Оператор+** — учащиеся из нескольких классов (глобальная работа; участникам уходит email)

Если указаны `room_ids`, требуется рассадка и нужны `supervisor_person_ids`.

### Видимость работ

| Роль | Доступ |
|------|--------|
| Учащийся | Свои работы; в рассадке — только своё место (`my_seat`) |
| Классный руководитель | Работы своего класса и глобальные с его классом; полная рассадка |
| Оператор+ | Все работы |

## Безопасность

- JWT **RS256** с полями `iss`, `aud`, `iat`, `nbf`, `exp`.
- Access-токен привязан к сессии (`sid`) - при отзыве сессии токен перестаёт работать.
- Refresh-токен хранится в БД только как SHA-256 хеш.
- При повторном использовании старого refresh-токена отзываются **все** сессии пользователя.
- Не более **5** активных сессий на пользователя.
- Смена пароля или роли отзывает все сессии пользователя.
- Rate limit:
  - **10 000** запросов/мин на IP
  - **3 000** запросов/мин на авторизованного пользователя
  - **60** попыток входа/мин на IP
- `/health` и документация не лимитируются.
- Задержка ответа при неверном пароле.
- Логируются только запросы с валидным access-токеном.

Лимиты настраиваются в `.env` (`RATE_LIMIT_*`, `LOGIN_RATE_LIMIT_*`).

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env
python -m uvicorn main:app --reload
```

Первый администратор:

```bash
python -m scripts.create_admin
```

Docker:

```bash
docker compose up -d --build
docker compose exec api python -m scripts.create_admin
```

Документация Swagger (только при `DEBUG=true`): `http://localhost:port/docs`
